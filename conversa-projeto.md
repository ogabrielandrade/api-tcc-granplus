# Notas para a apresentação

## Abertura

Nosso projeto tem como objetivo desenvolver um sistema de controle de estoque, pensado para organizar as informações e facilitar o acompanhamento dos produtos.

## Objetivos do projeto

O sistema permite cadastrar e gerenciar produtos, controlar entradas e saídas, registrar fornecedores, categorias, unidades de medida e locais, além de atualizar o estoque automaticamente e gerar relatórios e alertas.

## Requisitos principais

Entre os principais requisitos estão o cadastro de produtos, o controle de movimentações, a autenticação de usuários, a atualização automática do saldo e o armazenamento seguro das informações no banco de dados.

## Organização do banco de dados

Primeiro foi criado o banco de dados, com as tabelas separadas por função e ligadas por chaves estrangeiras. Também foram utilizadas triggers para atualizar o estoque automaticamente e índices para melhorar o desempenho das consultas.

## Ordem de desenvolvimento

A sequência do projeto foi: primeiro o banco de dados, depois a API e, por fim, a interface, que é a parte com a qual o usuário interage.

## Como explicar no slide

No slide, você pode resumir assim: Banco de dados -> API -> Interface.

## Fala curta para a apresentação

“Primeiro estruturamos o banco para organizar os dados. Depois desenvolvemos a API para fazer a comunicação entre o banco e o sistema. Por fim, criamos a interface, que é a parte visual usada pelo usuário.”
